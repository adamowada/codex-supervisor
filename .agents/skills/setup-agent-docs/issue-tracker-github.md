# GitHub Issue Tracker

Use GitHub Issues as the durable task tracker.

Prefer GitHub connector tools for reading and writing issues, pull requests, checks, comments, and CI logs. Use local `git` for branch, status, diff, and commit state.

Ready-for-agent issues need:

- current behavior or desired outcome;
- acceptance criteria;
- out-of-scope;
- verification commands;
- links to source-of-truth docs;
- HITL decisions already resolved.
