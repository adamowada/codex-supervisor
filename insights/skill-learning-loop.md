# Skill Learning Loop

Confidence: needs validation.

The supervisor should "learn" through versioned skills, not opaque model memory.

## Loop

1. Worker fails, stalls, or requires repeated correction.
2. Reviewer classifies the failure.
3. Supervisor writes an insight with provenance.
4. Skill maintainer proposes a skill change.
5. A golden task tests the old skill against the new skill.
6. Human or reviewer approves promotion.
7. Planning SQLite records the decision and linked artifact.
