# GitHub Issue Tracker

Use GitHub Issues as the publication and collaboration surface unless the repository explicitly
chooses GitHub Issues as the canonical queue. For `codex-supervisor` and spawned projects that use
planning SQLite, `plans/planning.sqlite3` remains canonical and GitHub issues mirror approved task
contracts.

Prefer GitHub connector tools for reading and writing issues, pull requests, checks, comments, and CI logs. Use local `git` for branch, status, diff, and commit state.

Ready-for-agent issues need:

- for public trackers, the first line `> *This was generated by AI during triage.*`;
- current behavior or desired outcome;
- acceptance criteria;
- out-of-scope;
- verification commands;
- links to source-of-truth docs;
- HITL decisions already resolved.
