# Local Markdown Issue Tracker

Use local markdown files when a repo has no remote issue tracker or the work is still private planning.

Recommended layout:

```text
docs/issues/
  ready-for-agent/
  ready-for-human/
  needs-info/
  wontfix/
```

Each issue file should contain the same contract as an AFK issue: goal, scope, acceptance criteria, verification, blockers, and source-of-truth links.
